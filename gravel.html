Gravel

Gravel is a C++ library with first-class Python bindings that quantifies how vulnerable road networks are to edge failures. It combines contraction-hierarchy routing with replacement-path fragility analysis to produce composite isolation scores for geographic regions — supporting disaster preparedness research, infrastructure planning, and transportation equity analysis.

v2.1 · released April 2026 · last commit 2026-04-10 · rhoekstr/gravel

The question routing libraries don't ask

Traditional routing answers what is the shortest path? Gravel answers a harder one: how does that path degrade when edges fail?

Why it matters

Disaster response plans assume road networks remain functional; they rarely do. Rural and mountain communities depend on single critical routes with no viable detours.

What Gravel produces

Quantitative fragility scores that make vulnerabilities comparable across counties, states, and regions — so "where is redundancy genuinely absent" is a ranked list, not a hunch.

Primary use cases

Ranking
Regions by vulnerability
Produce a fragility score for every county in a state or nation and rank them.
Scenario
Hazard footprint impact
Given a flood, wildfire, or landslide footprint, measure the resulting connectivity loss.
Progressive
Failure simulation
Degrade edges one at a time and plot the resulting isolation curve over the sequence.
Inter-region
Connectivity dependence
Quantify how reliant one county is on a neighbor's roads — and where that reliance concentrates.
Directional
Asymmetric vulnerability
Reveal that a coastal town's evacuation routes north are fine, while east is fragile.

Architecture

Six sub-libraries organized as a strict dependency DAG. Consumers link only what they need — a router-only tool never pulls in Eigen or Spectra.

gravel-core
ArrayGraph & primitives
Structure-of-arrays CSR graph, region assignment, basic types. No external dependencies.
gravel-ch
Contraction hierarchy
Compiled CH for O(log n) shortest-path queries, blocked-edge variants, bidirectional Dijkstra for verification.
gravel-simplify
Graph reduction
Degree-2 contraction shrinks 200K-node county graphs to ~14K with zero loss of shortest-path information.
gravel-fragility
Fragility analysis
Reverse-incremental SSSP, ensemble and uncertainty quantification, composite formula, progressive elimination.
gravel-geo
Geographic operators
GeoJSON boundary loading, point-in-polygon assignment, boundary-aware simplification, graph coarsening.
gravel-us
US-specific loaders
TIGER/Line for counties, states, CBSAs, places, urban areas. FIPS crosswalk built in.
Composite isolation formula

isolation_risk = 0.5 × disconnected_fraction + 0.3 × normalized_distance_inflation + 0.2 × coverage_gap. Weights chosen empirically so Swain County and Bryson City rank as intuitively more fragile than Asheville. Single source of truth in composite_formula.h.

What Gravel is not

Not turn-by-turn

Not a navigation library. Use OSRM or GraphHopper.

Not transit

Not a transit planner. Use OpenTripPlanner for GTFS and multi-modal.

Not dynamic

Contraction hierarchies assume a static edge set. No live traffic, no mutable graphs.
